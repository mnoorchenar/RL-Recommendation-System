About this System

This project implements a Deep Q-Network (DQN) recommender agent trained via offline reinforcement learning on synthetic MovieLens-style interaction data. The agent learns to predict which item will maximise long-term user satisfaction, not just immediate clicks.

  • 🧠 GRU State Encoder — encodes interaction history into a dense state vector
  • 🎯 DQN Policy — Q-network selects the highest-value item to recommend
  • 🔄 Experience Replay — 30K-transition buffer stabilises TD learning
  • 📊 Offline Training — learns from historical ratings (no live exploration needed)
  • 📈 OPE Evaluation — Precision@K, Recall@K, NDCG@K, Hit-Rate@K
  • 🎰 LinUCB Bandit — simpler contextual bandit baseline included

Architecture

  User Interaction History  [(item_id, reward), (item_id, reward), …]
              │
              ▼
  ┌─────────────────────────┐
  │  Item Embedding Layer   │  embed_dim = 16   (item index → dense vector)
  │  Reward Embedding Layer │  dim = 4          (0=skip, 1=liked)
  └───────────┬─────────────┘
              │  concat → input dim = 20
              ▼
  ┌─────────────────────────┐
  │     GRU Encoder         │  hidden_dim = 32, processes last 10 interactions
  └───────────┬─────────────┘
              │  state vector s ∈ ℝ³²
              ▼
  ┌─────────────────────────┐
  │     Q-Network (MLP)     │  Linear(32→128) → ReLU → Linear(128→128) → ReLU
  │                         │  → Linear(128→n_items)
  └───────────┬─────────────┘
              │  Q(s, a)  for all a ∈ {0 … n_items−1}
              ▼
         argmax_a Q(s, a)  →  Top-K Recommended Items
              │
              ▼
  ┌─────────────────────────┐
  │    Experience Replay    │  stores (s, a, r, s′, done) tuples
  └───────────┬─────────────┘
              │
              ▼
  TD Loss:  L = E[(r + γ · max_a′ Q̂(s′,a′) − Q(s,a))²]

RL Formulation

🤖

Agent

The DQN recommender that selects which item to suggest next

🌍

Environment

The user — their responses (likes / skips) form the reward signal

📋

State

GRU encoding of the last 10 user interactions (item + reward pairs)

🎬

Action

Which item to recommend from the 200-item catalog

⭐

Reward

+1 if user rating ≥ 4 (liked), 0 otherwise (skip / dislike)

🔮

Policy

ε-greedy during offline training; greedy argmax Q at inference

Metrics Reference

Metric Formula What it measures
Precision@K # relevant in top-K ÷ K Fraction of recommendations that are relevant
Recall@K # relevant in top-K ÷ total relevant Fraction of all relevant items that were surfaced
NDCG@K DCG@K ÷ IDCG@K Ranking quality — penalises relevant items ranked lower
Hit Rate@K 1 if any relevant item in top-K, else 0 Probability of surfacing at least one good recommendation
