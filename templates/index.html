What This System Does

This project implements a production-style RL-based ad recommendation engine — the same class of system powering ad serving at Meta, Google, LinkedIn, and TikTok. The agent learns to maximise long-term advertiser revenue while minimising user ad fatigue.

  • 🎯 DQN Policy — Q-network scores every ad in the catalogue per user session
  • 🧠 GRU State Encoder — encodes recent impression history + user demographics
  • 📈 CTR/CVR Predictor — neural model gives interpretable click & conversion estimates
  • ⚖️ Composite Reward — click + conversion bonus − fatigue penalty + revenue signal
  • 😴 Ad Fatigue Modelling — CTR decays after repeated exposure to same ad
  • 💰 Bid-Aware Ranking — higher bid-price ads get a revenue signal boost
  • 📊 Offline Evaluation — CTR, CVR, eCPM, ROAS, NDCG@K, Hit-Rate@K
  • 🎰 LinUCB Bandit — exploration baseline (Thompson Sampling-style)

RL Formulation for Advertising

🤖

Agent

The ad serving system — decides which ad to show each user in each session

🌍

Environment

The user population — click, convert, or skip drives the reward signal

📋

State

52-dim: GRU(ad_history) + user_demographics + temporal context

📢

Action

Which ad to serve from the 100-ad catalogue (+ bid price)

💎

Reward

+1 click · +3 convert · −0.2 fatigue · +0.001 bid_price (revenue signal)

🔍

Exploration

ε-greedy decay during offline training; greedy argmax Q at serving

Industry Context — Why This Matters

Real-Time Bidding (RTB) systems must rank millions of ads for each page load in <100ms. RL is used to optimise long-term ROAS rather than greedy CTR maximisation, which rewards clickbait.

Key papers behind this project:

  • DRN: Deep Reinforcement Learning for News Recommendation (Zheng et al., 2018)
  • Reinforcement Learning for Slate-based Recommenders (Ie et al., Google, 2019)
  • Contextual Bandits for Ad Click-Through Rate (Li et al., Yahoo, 2010)
  • Offline RL for Recommender Systems (Levine et al., 2020)

Reward shaping matters more than model architecture. Raw CTR optimisation leads to clickbait. The composite reward here penalises:

  • 😴 Ad fatigue — same ad shown too many times to same user
  • 🚫 Category monopoly — diversity penalty keeps feed varied
  • 📉 Low-value clicks — bid-weighted signal prioritises high-ROAS ads

Training Pipeline

📦

Data

300 users × 100 ads → ~19K logged impressions converted to (s, a, r, s′) transitions

🔄

Replay Buffer

30K-capacity deque. Random mini-batch sampling breaks temporal correlations

📉

TD Learning

Smooth-L1 (Huber) loss on Bellman targets. Target network updated every 200 steps

⚡

Optimizer

Adam with lr=1e-3. Gradient clipping at 1.0 for training stability

🎲

Exploration

ε decays 0.995× per step from 1.0 → 0.05 minimum

🎯

Reward

Composite: click(+1) + convert(+3) − fatigue(−0.2) + revenue(+0.001×bid)
